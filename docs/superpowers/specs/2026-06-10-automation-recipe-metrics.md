# 自动化动作配方准入指标

## 目标

本文档定义哪些手机操作可以进入自动化页，以及动作配方可以从哪些来源生成。它作为后续实现、测试和产品判断的核心量化指标。

自动化页不应该成为普通历史记录页。只有能被稳定复用、可验证、风险可控的操作，才应该沉淀为动作配方。

## 核心概念

- **动作配方**：一组带验证点的工具调用步骤，用于快速完成重复手机操作。
- **Boost**：命中白名单动作配方后的快速执行状态。
- **候选配方**：系统从历史任务或 Agent 成功路径中提取出的待审核动作。
- **已启用配方**：用户审核通过、允许在自动化页中管理的动作。
- **白名单配方**：允许自动 Boost 执行的已启用配方。

## 准入原则

一个操作要进入自动化页，必须同时满足以下条件：

1. **可重复**
   - 用户可能第二次、第三次执行相同或相似操作。
   - 操作不是一次性排障、一次性查询或临时探索。

2. **边界清楚**
   - 有明确起点，例如打开某个 App、从当前页面开始、从系统设置页开始。
   - 有明确终点，例如到达某个页面、打开某个开关所在页、出现某个 UI 元素。

3. **可验证**
   - 成功条件能通过包名、Activity、页面文字、content-desc、UI 节点、截图状态或 HTTP 状态判断。
   - 不能只依赖模型主观判断“看起来完成了”。

4. **低风险或可降级确认**
   - 导航、打开页面、普通点击适合自动执行。
   - 输入、授权变更、发送、删除、支付等操作必须降级为确认后执行或禁止自动执行。

5. **参数可分离**
   - 可变内容不能被硬编码进长期配方。
   - 例如“进入某联系人聊天页”可以成为配方，“发送这次的消息内容”应作为运行时参数或确认动作。

6. **具备稳定锚点**
   - 优先使用 text、content-desc、class、父节点摘要、Activity 等锚点。
   - 坐标只能作为兜底，不能作为唯一定位方式进入白名单。

## 量化准入指标

每个候选配方应计算以下指标。第一版可以手动或半自动评估，后续可自动化。

| 指标 | 说明 | 建议阈值 |
| --- | --- | --- |
| 重复价值 | 用户是否可能重复执行 | 至少出现过 2 次相似意图，或用户主动保存 |
| 边界清晰度 | 起点和终点是否明确 | 起点、终点都能写成结构化条件 |
| 可验证性 | 是否能自动判断成功 | 至少 1 个结束验证条件，关键步骤有轻量验证 |
| 风险等级 | 是否可能造成不可逆影响 | 白名单自动执行仅允许低风险 |
| 锚点稳定性 | 元素定位是否稳定 | 至少 1 个非坐标锚点 |
| 参数分离度 | 可变内容是否独立 | 敏感/动态输入不能写入固定步骤 |
| 历史成功率 | 已记录路径是否稳定 | 自动白名单建议成功率 >= 80% |
| 失败可恢复性 | 失败后能否安全停止并回退 Agent | 必须支持失败停止和回退 |

## 风险等级

### 低风险

可以进入自动化页，也可以在用户白名单后自动 Boost。

- 打开 App。
- 进入某个设置页面。
- 点击导航按钮或 Tab。
- 展开菜单。
- 返回上一级。
- 打开搜索页但不提交敏感内容。

### 中风险

可以进入自动化页，但默认需要确认；用户可显式加入白名单时，仍应保留步骤级验证。

- 切换普通设置项。
- 填写非敏感表单。
- 打开权限设置页但不直接授权。
- 执行可能改变 App 状态但可撤销的操作。

### 高风险

第一版不允许自动 Boost。可以记录为候选，但只能作为 Agent 上下文或确认后辅助。

- 支付、转账、下单。
- 删除文件、删除聊天、清空数据。
- 发送消息、发帖、评论、拨打电话。
- 授予高敏权限。
- 输入密码、验证码、API Key、Token。
- 任何不可逆或对外部产生影响的提交动作。

## 不应进入自动化页的操作

- 一次性问答、一次性排障。
- 无法定义成功条件的开放式探索。
- 页面高度动态且缺少稳定 UI 锚点的操作。
- 只依赖截图视觉猜测、没有 UI 树锚点的操作。
- 需要保存敏感输入才能复用的操作。
- 连续失败且无法自动修复的路径。

## 动作来源

### 1. Agent 成功执行后自动候选

Agent 第一次正常探索，任务成功后系统提取工具调用轨迹，生成候选配方。

适用场景：

- 用户自然完成一次任务后，希望下次更快。
- 工具调用链路完整，有明确成功输出。
- 已有 task_id 可以关联一次任务内的动作。

准入要求：

- 必须能识别开始工具、关键动作、结束验证。
- 必须剔除截图观察、重复读取 UI 树等非必要步骤，保留可执行骨架。
- 必须标记风险等级，不能直接进入自动白名单。

### 2. 用户手动录制

用户点击“录制”，自己操作一遍，系统记录 UI 节点、点击路径和页面变化。

第一版状态：

- 保留为方案和数据模型扩展点。
- 暂不作为首批实现范围。

保留原因：

- 对用户熟悉但 Agent 不容易理解的操作更直接。
- 后续可以用来校正 Agent 生成的配方。
- 可以为无模型或弱模型场景提供快速路径。

准入要求：

- 录制时必须显示状态。
- 录制结束后必须进入审核页，不能直接白名单。
- 录制内容必须做敏感输入脱敏。

### 3. 从历史任务提取

从 AgentTask 和 MCP 工具调用历史中提取成功路径，生成候选配方。

适用场景：

- 用户已经多次完成相似任务。
- 历史里存在可复用工具调用序列。
- 需要批量发现高重复价值操作。

准入要求：

- 只从完成状态任务中提取。
- 候选配方必须附带来源任务和最后验证信息。
- 多次相似历史应合并为同一个配方的候选版本。

## 相似操作去重指标

相似操作需要去重。否则自动化页会快速堆积多个“看起来不同、实际目标相同”的候选配方，增加用户审核和白名单管理成本。

去重应发生在候选配方阶段，而不是用户已经审核启用之后才处理。第一版做轻量规则去重，不做复杂语义聚类。

### 应合并的情况

满足以下大部分条件时，应合并为同一个候选配方或同一配方的新版本：

- 用户意图归一化后相近，例如“打开无障碍设置”和“进入辅助功能权限页”。
- 目标 package 相同。
- 结束条件相同或高度相近。
- 关键 UI 锚点相同，例如目标页面标题、目标按钮文本、content-desc。
- 工具调用骨架相近，例如 `app.open -> ui.click_text -> ui.click_text`。
- 风险等级相同。

### 不应合并的情况

以下情况必须保留为独立配方：

- 结束条件不同，即使中间路径相似。
- 风险等级不同，例如一个只打开页面，另一个会切换开关。
- 参数模型不同，例如一个只是进入搜索页，另一个会提交搜索词。
- 目标 App 不同。
- 涉及不同账号、联系人、文件或外部对象。
- 一个配方包含高风险动作，另一个不包含。

### 去重评分建议

第一版可以用规则评分：

| 维度 | 权重建议 |
| --- | --- |
| 目标 package 相同 | 高 |
| 结束条件相同 | 高 |
| 关键 UI 锚点相同 | 高 |
| 归一化意图相近 | 中 |
| 步骤工具序列相近 | 中 |
| Activity 相同 | 中 |
| 坐标相近 | 低 |

坐标相近不能单独作为去重依据。坐标只说明点击位置相似，不说明语义相同。

### 合并结果

合并时不应丢弃信息。系统应保留：

- 所有来源任务 ID。
- 各版本步骤。
- 每个版本的成功率和失败原因。
- 最近一次成功版本。
- 用户当前启用的版本。

如果新候选和旧配方相似但更稳定，应生成“可升级版本”，由用户审核或由失败修复流程推荐升级。

## 配方进入状态

配方从候选到自动执行，需要经过以下状态：

```text
候选配方
  -> 用户审核通过
已启用配方
  -> 用户加入 App 或配方白名单
白名单配方
  -> 命中意图和上下文
Boost 执行
  -> 成功：更新成功率和耗时
  -> 失败：取消 Boost，降级或禁用，回退 Agent
```

## 自动白名单条件

配方可以被加入自动白名单，但必须满足：

- 风险等级为低风险，或用户明确允许中风险自动执行。
- 至少有一个稳定的非坐标定位锚点。
- 有明确结束验证。
- 最近失败次数未超过阈值。
- 不包含敏感输入。
- 不包含高风险提交动作。

## 失败处理指标

一次 Boost 失败时，系统必须做到：

- 立即取消 `boosting` 状态。
- 悬浮窗显示失败并进入回退状态。
- 停止继续执行后续步骤。
- 将失败步骤、当前包名、Activity、屏幕指纹写入记录。
- 回退给 Agent 正常探索。
- Agent 成功后生成修复候选，而不是盲目覆盖原配方。

连续失败处理建议：

- 1 次失败：降低该配方置信度。
- 2 次失败：暂停自动白名单，仍保留手动执行。
- 3 次失败：禁用配方，等待用户审核或 Agent 修复。

## 后续评估口径

实现后应重点衡量：

- 命中率：相似任务中成功识别配方的比例。
- Boost 成功率：命中后直接完成的比例。
- 平均节省时间：普通 Agent 执行耗时减去 Boost 执行耗时。
- 误触率：执行了不符合用户目标动作的比例。
- 回退成功率：Boost 失败后 Agent 是否能完成任务。
- 用户保留率：用户保留、禁用、删除配方的比例。
