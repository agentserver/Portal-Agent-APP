# Claude Code Android App 工作进展总结

本工作的内容是一个 Android 端的手机操作执行体，作为 AgentServer 和 Loom 的手机侧接入端，使两个项目能够读取并调用手机的摄像头拍照、屏幕截图、无障碍 UI 点击输入等原生能力，实现从感知到操作的完整闭环。

基础能力方面，在 Termux 内通过 proot-distro 搭建 Ubuntu 容器运行 Claude Code，构建了 13 个 MCP 工具（截屏、无障碍 UI 控制、拍照、文件读写、App 控制、设备状态查询），通过 localhost 的 MCP HTTP Server 和 ApiHttpBridge 将手机能力暴露给容器内的模型调用。AgentServer 远程接入通过 agentserver 长驻进程建立 WebSocket tunnel，使上游能将任务派发到手机执行，并以 wrapper 拦截 stream-json IO 流实现对话记录与回放。Loom 以独立 addon 包集成，提供 observer/driver/slave 多角色框架的配置与启停管理，与 AgentServer 共享 Ubuntu 运行时但保持进程隔离。后期接入 Codex 作为第二模型后端，实现单环境双用户架构。

当前研究重心在自动化 Boost：将 Agent 首次探索手机的成功操作路径沉淀为可复用的动作配方，后续遇到相似任务直接走快速路径执行。围绕这一目标，定义了配方准入指标体系（可重复、边界清楚、可验证、低风险、参数可分离、具备稳定 UI 锚点），从 MCP 工具调用轨迹中自动提取候选配方并做相似去重与版本合并，实现 Boost 调度、白名单管理和连续失败降级，失败后回退 Agent 正常探索并生成修复候选。目前正在规划什么能力可以被纳入已发现能力库，以及如何进一步提升配方命中与执行稳定性。
